Tareas

El corazón operativo del sistema. Acá vive el trabajo diario del equipo: cada tarea tiene un responsable, una fecha de entrega, prioridad y estado.

Estados de una tarea

  • Pendiente — todavía no se empezó.
  • En Progreso — alguien está trabajando en ella ahora.
  • Completada — finalizada. El admin recibe una notificación automática.

Filtros disponibles

Podés filtrar por fecha (hoy, esta semana, este mes), estado, prioridad, cliente y proyecto. Los filtros se combinan entre sí para encontrar exactamente lo que necesitás.

Los usuarios con rol user solo ven las tareas asignadas a ellos. Los admins ven todas.

Notas del técnico

Cada tarea tiene un campo de notas internas donde el responsable puede documentar qué hizo, qué problema encontró o cómo lo resolvió. Muy útil para auditorías y para que el equipo no pierda contexto.

Control de Tareas

El panel principal del sistema. Muestra de un vistazo cuántas tareas están pendientes, en progreso y completadas.

Hacé clic en cualquier tarjeta de estadística para ir directo a las tareas filtradas por ese estado.

Calendario integrado

Debajo de las estadísticas hay un calendario mensual con todas las fechas de entrega de las tareas. Cada día con tareas muestra puntos de color según el estado (naranja = pendiente, azul = en progreso, verde = completada). Hacé clic en un día para filtrar automáticamente la sección Tareas y ver solo lo de esa fecha.

Proyectos

Un proyecto agrupa tareas relacionadas bajo un mismo objetivo. Ideal para trabajos con varias etapas, como el desarrollo de un sitio web o el lanzamiento de un producto.

Cómo trabajar con proyectos

  1. Creás el proyecto con nombre, cliente, fechas de inicio y entrega, y un color identificador.
  2. Agregás tareas desde el botón "Nueva Tarea" dentro del detalle del proyecto. Así quedan automáticamente vinculadas.
  3. El progreso se calcula solo: a medida que completás tareas, la barra del proyecto sube. Si tenés 5 tareas y completás 3, el proyecto marca 60%.
  4. Seguís el avance desde la vista de Proyectos (panorama general) o desde Tareas (trabajo del día a día).
También podés asignar un proyecto a cualquier tarea existente desde el modal de edición de tareas, en el campo "Proyecto".

Reuniones

Gestioná todas las reuniones del equipo: tanto virtuales (con link) como presenciales (con dirección). Cada usuario ve solo sus propias reuniones; los admins ven todas.

Campos principales

  • Tipo: Virtual (activa el campo Link) o Presencial (activa el campo Dirección).
  • Temas a tratar: listá los puntos de la agenda antes de la reunión.
  • Notas/Resultados: completá después de la reunión con conclusiones y próximos pasos.
  • Duración: desde 15 minutos hasta 3 horas.

Vista Calendario

Dentro de la sección hay un calendario propio que muestra las reuniones del mes con chips de color (azul = virtual, verde = presencial). Hacé clic en un día para ver el detalle de todas las reuniones de esa fecha.

Estados

  • Programada → podés marcarla como Realizada o Cancelada.
  • Realizada → ideal para completar las notas de resultado.
  • Cancelada → se muestra opaca para distinguirla visualmente.

Presupuestos

Control de cotizaciones enviadas a clientes y prospectos. Cada presupuesto tiene un flujo de estados que refleja en qué punto está la negociación.

Flujo de estados

  • Borrador → todavía no se envió. Botón para marcar como Enviado.
  • Enviado → esperando respuesta. Botones Aceptado / Rechazado / Vencido.
  • Aceptado → botón para Convertir a Ingreso en Finanzas con un solo clic.
  • Rechazado / Vencido → botón para Reprogramar si querés reactivarlo.

PDF adjunto

Podés adjuntar el PDF del presupuesto directamente en el sistema (hasta 15 MB). Aparece como botón "Ver PDF" en la card, accesible para todo el equipo.

Filtros avanzados

Buscá por título o cliente, filtrá por estado, ordená por fecha o monto, y acotá por rango de fechas y monto mínimo/máximo.

Clientes

Directorio de todos tus clientes activos. Cada cliente puede tener logo, email, teléfono, Instagram y una descripción corta.

Los clientes aparecen como opciones en Tareas, Presupuestos, Finanzas y Reuniones — siempre que cargues algo nuevo, podés vincularlo a un cliente existente.

Solo los admins pueden crear, editar y eliminar clientes. Todos pueden verlos.

Finanzas

Registro de todos los ingresos y gastos del negocio. No es contabilidad formal, es control operativo: saber en todo momento si el negocio genera más de lo que gasta.

Ingresos vs Gastos

Cada movimiento se clasifica con una categoría. Los ingresos pueden vincularse a un cliente. El balance se calcula automáticamente y cambia de color (azul = positivo, rojo = negativo).

Categorías personalizables

Usá el botón Categorías para agregar las tuyas propias. Las predeterminadas no se pueden eliminar, pero podés agregar todas las que necesites.

Gráficas

Hacé clic en Ver Gráficas para ver 4 visualizaciones:

  • Evolución mensual — barras comparando ingresos vs gastos en los últimos 12 meses.
  • Gastos por categoría — dona mostrando en qué se va el dinero.
  • Ingresos por categoría — dona mostrando de dónde viene el dinero.
  • Balance acumulado — línea de tiempo del balance neto.
Cambiá el filtro de período (Este mes / Este año / Todo el tiempo) para que las gráficas y los totales se actualicen automáticamente.

Centro de Prospección

Es el tablero de ventas del negocio. Muestra todas las oportunidades comerciales organizadas en columnas según en qué etapa del proceso están. Funciona como un Kanban.

Las 6 etapas

  1. Prospecto — alguien mostró interés, todavía no hablaste.
  2. Contactado — ya tuviste una primera conversación.
  3. Propuesta — le enviaste un presupuesto o propuesta.
  4. Negociando — están revisando condiciones o pidiendo cambios.
  5. Ganado — aceptaron. ¡Cerraste la venta!
  6. Perdido — no se concretó.

Cómo mover un deal

Simplemente arrastrá la card de una columna a la siguiente. El cambio se guarda automáticamente.

Cuando un deal pasa a Ganado, aparece el botón para convertirlo directamente en un ingreso en Finanzas, sin tener que cargar nada dos veces.

Valor ponderado

El stat "Valor Potencial" multiplica el monto de cada deal por su probabilidad de cierre. Así tenés una estimación realista de cuánto podría ingresar, no solo la suma bruta.

Base de Conocimiento

Un repositorio de artículos, guías y documentación interna para el equipo. Todo el equipo puede leer, solo los admins pueden escribir.

Para qué usarla

  • Documentar procesos internos: cómo se onboardea un cliente, cómo se entrega un proyecto.
  • FAQ con preguntas frecuentes del equipo.
  • Plantillas de mensajes, propuestas o respuestas tipo.
  • Recursos útiles: links, guías de herramientas, accesos.

Formato de los artículos

El contenido soporta formato básico:

  • # Título grande y ## Subtítulo
  • **texto en negrita**
  • - item para listas
  • --- para separadores

Los artículos se organizan por categorías que aparecen en el panel izquierdo con el conteo de artículos de cada una.

Mi Equipo

Gestión del equipo. Solo los admins pueden crear, editar o eliminar usuarios.

Roles

  • Admin — acceso completo a todo el sistema. Ve todas las tareas, puede gestionar usuarios, clientes, finanzas, presupuestos y configuración.
  • User — acceso limitado. Ve solo sus tareas asignadas, puede registrar reuniones propias, leer la Base de Conocimiento y usar el Calendario.
No podés eliminar tu propio usuario. Siempre debe quedar al menos un admin activo en el sistema.

Registro

Historial completo de todo lo que pasa en el sistema. Solo visible para admins.

Cada acción queda registrada con: quién la hizo, qué fue, cuándo y con qué datos. Útil para auditorías, para resolver conflictos y para entender qué cambió y cuándo.

Podés buscar por usuario, tipo de acción o cualquier detalle usando el buscador en tiempo real.

Si algo "desapareció" o "cambió solo", el Registro es el primer lugar donde buscar. Siempre queda la huella de quién hizo qué.
